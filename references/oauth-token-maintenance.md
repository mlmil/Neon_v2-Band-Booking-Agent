# Google OAuth Token Maintenance

## Why Tokens Expire

The Google Cloud project (`stellar-vista-486501-s2`) is in **Testing** mode — never published to Production. Google's policy for unverified testing-mode apps is:
- **Refresh tokens expire after 7 days**
- The OAuth consent screen requires re-authorization when the token dies
- There is no way to get permanent tokens without publishing the app

**Bottom line**: the band calendar OAuth token will need refreshing roughly every week. This is a known Google limitation, not a bug in the scripts.

## Token File

```
~/.hermes/neon_oauth_token.json
```

This file is auto-generated by the OAuth flow. It contains the access token, refresh token, and expiry info. When expired, the refresh attempt yields:

```
invalid_grant: Token has been expired or revoked.
```

## Re-Auth Workflow

When the user wants to re-authenticate:

### Step 1: Generate the OOB URL

Run the helper script:
```bash
python3 ~/.hermes/skills/Neon_v1/scripts/generate_oob_auth_url.py
```

This prints a Google OAuth URL. Present it to the user with clear instructions.

### Step 2: User Opens in Incognito Browser

Tell the user:
1. Open in an **incognito/private browser window**
2. Sign in as `neonblondevc@gmail.com`
3. Approve the "View your calendars" permission
4. Copy the **code** shown on the resulting white page

### Step 3: Exchange the Code for a Token

```python
import json
from google_auth_oauthlib.flow import InstalledAppFlow
from pathlib import Path

SCOPES = ['https://www.googleapis.com/auth/calendar']
TOKEN_FILE = Path.home() / '.hermes' / 'neon_oauth_token.json'

flow = InstalledAppFlow.from_client_secrets_file(
    '/Users/studio_hub/tools-registry/mcps/google_workspace_mcp/client_secret.json',
    SCOPES,
    redirect_uri='urn:ietf:wg:oauth:2.0:oob'
)
flow.fetch_token(code='<paste-code-here>')
creds = flow.credentials
TOKEN_FILE.write_text(creds.to_json())
print(f"Token saved to {TOKEN_FILE}")
```

### Step 4: Verify

```bash
python3 ~/.hermes/skills/Neon_v1/scripts/neon_calendar_oauth.py
```

If the calendar events print correctly, the token is working.

## Known Pitfalls

- **VADER not mounted**: The `generate_oob_auth_url.py` script checks for VADER first. If not mounted, prompt the user to mount it (macOS: open Finder → VADER under Locations).
- **client_secret file missing**: If the VADER path is wrong, search for new copies:
  ```bash
  find /Volumes/VADER -name "client_secret*.json" 2>/dev/null
  ```
- **Stale feed cache**: After a successful re-auth, the Google Calendar API responds immediately. But the public iCal feed (Mark's calendar) may cache for 30-60 seconds.
